Security Policy

## Supported Versions

TokenGuard is currently in its initial `0.0.x` release line. Security fixes
are provided for the latest published release only.

| Version | Supported |
| --- | --- |
| Latest `0.0.x` release | :white_check_mark: |
| Earlier releases | :x: |
| Unreleased development builds | :x: |

## Reporting a Vulnerability

Please do **not** open a public issue for a suspected security vulnerability.

Instead, use GitHub's private reporting flow for this repository:

1. Go to the repository's **Security** tab.
2. Select **Report a vulnerability**.
3. Include a clear description, affected TokenGuard version, reproduction
   steps or proof of concept, impact, and any suggested mitigation.

Do not include API keys, access tokens, personal data, or other sensitive
information in your report. If sensitive evidence is necessary, say so in the
report and wait for a maintainer to provide a secure way to share it.

## What to Expect

We aim to acknowledge valid reports within 3 business days and provide a
status update at least every 7 days while investigating. We will assess the
report, determine the affected versions and severity, and work on a fix or
mitigation.

If a report is accepted, we will coordinate a release and credit the reporter
where requested. If it is declined, we will explain why. Please allow time for
users to update before disclosing technical details publicly; we will
coordinate the disclosure timing with the reporter whenever possible.

## Scope

This policy covers the TokenGuard dashboard, local daemon, CLI, plugins, and
official release artifacts in this repository. Third-party services and
dependencies should also be reported to their respective maintainers when
appropriate.
